# 要件定義書

## 1. プロジェクト概要
**プロジェクト名**: 価値の序列アプリ (Value Hierarchy App)
**目的**: ユーザーが自身の「価値の序列（Value Hierarchy）」を明確にし、自己理解を深めるためのワークをWebアプリとして提供する。
**ターゲット**: 自己分析、自己啓発、人生の目的発見に関心があるユーザー。

## 2. 機能要件

### 2.1 質問回答機能
- **8つのカテゴリー**: 以下の8つの領域について質問を行う。
  1. 空間 (Space)
  2. 時間 (Time)
  3. エネルギー (Energy)
  4. お金 (Money)
  5. 想像 (Imagination)
  6. 話題 (Talk)
  7. 学習 (Knowledge)
  8. 自己管理 (Self-Management)
- **回答形式**: 各カテゴリーにつき、上位3つの要素をテキストで入力する。
- **目的の深掘り**: 2番目の回答に対してのみ、「それは何のため？」という目的（Purpose）を入力する欄を設ける。
- **ガイダンス**: 各質問の意図を説明するガイダンスと、具体的な回答例を表示・非表示切り替え可能にする。

### 2.2 グルーピング機能（まとめ）
- **一覧表示**: 全8カテゴリー×3回答（計24回答）を一覧表示する。
- **グループ作成**: ユーザーが類似した回答を選択し、任意のグループ名（ラベル）を付けてグループ化できる。
- **色分け**: グループごとに自動または手動で色を割り当て、視覚的に区別する。

### 2.3 分析・可視化機能
- **グラフ表示**: 作成されたグループの割合（回答数ベース）を、バーチャートおよび円グラフで可視化する。
- **インサイト入力**: 各グループ（価値観の塊）に対して、「なぜそれが重要なのか」「どう活かすか」といった考察をテキストで入力・保存できる。

### 2.4 データ管理機能
- **自動保存**: 入力内容はブラウザのローカルストレージ（localStorage）に即時保存され、再訪問時に復元される。
- **リセット**: 全データを消去し、最初からやり直す機能を提供する。
- **エクスポート**: 入力データおよび分析結果を、JSON形式またはテキスト形式でファイルとしてダウンロードできる。

## 3. 非機能要件

### 3.1 UI/UX
- **デザインコンセプト**: 「静かな内省の空間」。集中力を高める落ち着いた配色（ダークブルー、グレー、ゴールド）と、余白を活かしたミニマルなデザイン。
- **レスポンシブ**: PC、タブレット、スマートフォンいずれのデバイスでも快適に操作できること。
- **アニメーション**: 画面遷移や操作時に滑らかなアニメーションを取り入れ、心地よい体験を提供する。

### 3.2 技術制約
- **プラットフォーム**: モダンブラウザ（Chrome, Edge, Safari, Firefox）で動作するWebアプリケーション。
- **データ保存**: サーバーサイドデータベースは使用せず、クライアントサイド（localStorage）のみで完結させる（プライバシー保護と手軽さのため）。
- **オフライン動作**: 初回ロード後はオフラインでも動作可能であることが望ましい（PWA化は将来的な検討事項）。

### 3.3 著作権対応
- 元となるワークブックの具体的な文言や表現をそのまま使用せず、一般的な自己分析ツールの表現にリライトする。
